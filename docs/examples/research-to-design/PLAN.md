# PLAN.md

## Objective

Produce a clearer client intake workflow grounded in explicit findings about
the current process.

## Phases

### Phase 1. Investigate Current Intake

Goals:

- identify current intake channels
- identify recurring friction points
- identify meaningful constraints and unknowns

Primary outputs:

- `artifacts/research/landscape.md`

### Phase 2. Synthesize Findings

Goals:

- condense the landscape into a smaller set of findings
- identify tensions that matter for design
- decide which questions remain open

Primary outputs:

- `artifacts/research/synthesis.md`

### Phase 3. Propose Revised Workflow

Goals:

- define a revised intake workflow
- assign role responsibilities across the workflow
- record at least one durable structural decision

Primary outputs:

- `artifacts/design/intake-workflow.md`
- `history/decisions/0001-separate-triage-from-full-intake.md`

## Intentional Deferrals

This example does not try to define:

- software implementation details
- staffing plans
- metrics instrumentation
- automation tooling
- compliance policy text

Those can be added later if the domain implementation needs them.
