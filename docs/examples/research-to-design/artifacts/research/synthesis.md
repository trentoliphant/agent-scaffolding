# Intake Findings Synthesis

## Purpose

Turn the intake landscape into a smaller set of conclusions that can guide
workflow design.

## Source Artifacts

- `artifacts/research/landscape.md`

## Key Conclusions

### 1. Early triage and full intake should not be treated as the same step

The current process asks one loose intake step to do too much.
Urgency assessment, information collection, and service assignment have
different needs and should be separated.

### 2. The workflow needs a small shared intake vocabulary

The current process uses inconsistent language for stages and handoffs.
The design should introduce a small number of clear states that every actor can
use consistently.

### 3. The first contact path must work with incomplete information

Some requests arrive with too little context for full routing.
The design should support an early response path that is useful even when the
intake packet is incomplete.

## Tensions

- the team wants consistent intake, but not every channel can require the same
  information immediately
- the team wants faster triage, but not at the cost of losing important case
  context

## Recommended Next Step

Design a workflow that separates:

- initial receipt
- early triage
- full intake completion
- assignment to the service team

## Open Questions

- Should urgent cases create a different workflow branch from the start?
- Which actor owns requesting missing information after early triage?
