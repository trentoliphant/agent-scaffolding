# Task 3.1 — Demo: Frame export endpoint

This is a fictional spec used by the v2 prototype's demo ledger so
`harness.py brief` has something real to assemble. It is not scheduled work.

## Objective

Add `GET /v1/frames/{id}/export` returning a portable single-Frame export
document, per the exportability invariant.

## Dependencies

None (demo).

## In Scope

- Export endpoint in Hub Core, both profiles, capability-gated.
- Deterministic, unique export filenames.
- Audit event `frame.exported`.

## Out of Scope

- Import (covered by manual Frame import, task 2.5).
- Bulk export, scheduling, Enterprise policy controls.

## Verification

- Hub test suite passes; new endpoint covered by contract-conformance tests
  against `contracts/example-api.md`.
- Round-trip check: exported document re-imports via the 2.5 path.

## Acceptance Criteria

- Endpoint present in both profiles with capability negotiation.
- Export document validates against the artifact manifest schema where
  applicable.
- Audit event recorded with scope and actor.
