# Roadmap

This document lists likely next improvements for the repository.

It is a planning aid, not a durable definition of the scaffold. Items here may change as the repository evolves.

## Near Term

1. Add a copyable starter template for builders
2. Expand examples to show a more complete workflow
3. Add more decision records as interpretation and scope choices are made
4. Improve cross-linking between guides, examples, and scaffold definitions

## Medium Term

1. Define what a minimal reusable scaffold package should contain
2. Clarify where orchestration guidance should stop being generic
3. Decide whether structured state formats are worth the added complexity
4. Add guide material for applying the scaffold incrementally to an existing project

## Watch Items

These are not immediate commitments, but they should remain visible:

1. Avoid over-formalizing the scaffold before real usage pressures justify it
2. Avoid turning examples into implicit universal definitions
3. Keep contributor-facing refinement from overwhelming builder-facing usability

## Completion Rule

When a roadmap item becomes the active frontier of work, reflect that in `state/current.md`.

When a roadmap item turns into a durable structural choice, record that in `history/decisions/`.
